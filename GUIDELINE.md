# M몰 기획전 빌더 — Agent 지침서

> 이 문서는 기획전 컨셉 입력 → 자동 초안 생성 과정의 판단 기준과 이미지 생성 규칙을 정의합니다.
> 전반적인 디자인 구조·모듈 구성은 현재 빌더(App.jsx / 기획전_빌더.jsx)를 기준으로 합니다.

---

## 1. 콘셉트 문구 해석 기준

입력된 콘셉트 문구를 아래 우선순위 순서로 파싱하여 기획전 타입을 결정합니다.

### 1-1. 시즌 키워드
| 인식 키워드 | 처리 방향 |
|---|---|
| 여름 / 썸머 / SUMMER | SUMMER COLLECTION 배너, 여름 시즌을 위한 셀렉션 |
| 겨울 / 윈터 / WINTER | WINTER COLLECTION 배너 |
| 봄 / 스프링 / SPRING | SPRING COLLECTION 배너 |
| 가을 / 오텀 / AUTUMN | AUTUMN COLLECTION 배너 |
| 시즌 (계절 명시 없음) | SEASON COLLECTION으로 처리 |

> ✏️ 추가 시즌 키워드나 영문 변형은 여기에 보완 예정

### 1-2. 혜택 키워드
| 인식 패턴 | 처리 방향 |
|---|---|
| `N%` / `최대 N%` | 할인 타입 → 히어로 타이틀에 `최대 N%` 반영 |
| `N,000원` / `N원` | 쿠폰 타입 → 쿠폰 모듈 금액 자동 입력 |
| `무료 배송` / `배송 무료` | 혜택 아이템에 🚚 무료 배송 추가 |
| `적립` / `포인트` | 혜택 아이템에 💎 포인트 적립 추가 |
| 혜택 구분자 | `/` (슬래시)로 구분 입력 → 각각 파싱 |

### 1-3. 타깃 키워드
| 인식 패턴 | 적용 대상 문구 |
|---|---|
| 첫 구매 / 신규 / 가입 | 첫 구매 고객 → Welcome M·MALL 배너 |
| VIP / 프리미엄 고객 | 해당 문구 그대로 사용 |
| 전체 고객 | 전체 고객 |
| (명시 없음) | 기본값: 전체 고객 |

### 1-4. 이벤트 유형 키워드
| 인식 패턴 | 처리 방향 |
|---|---|
| 플래시 / 타임 / 긴급 / 한정 / 오늘만 | 24H FLASH SALE 배너 |
| 세일 / 할인 / 특가 / 파격 | SPECIAL SALE 배너 |
| (분류 불가) | 컨셉 문구 내 `XX 기획전` 명사 추출 → 타이틀 사용 |

### 1-5. 카테고리 키워드
콘셉트 문구에 아래 카테고리가 2개 이상 포함되면 상품리스트 탭이 자동 생성됩니다.

`패션` / `뷰티` / `리빙` / `푸드` / `가전` / `디지털` / `스포츠` / `라이프스타일`

## 2. 텍스트 톤앤매너

- **길이**: 타이틀 최대 2줄 / 한 줄 12자 이하 권장
- **길이**: 타이틀 최대 2줄 / 한 줄 12자 이하 권장
- **톤**: 세련되고 간결하게. 과도한 형용사, 느낌표 남용 지양
- **줄바꿈**: `\n` 기준 2줄 구성 (히어로 타이틀·텍스트 섹션 공통)
- **영문 서브타이틀**: 대문자 고정 (예: `SUMMER COLLECTION`, `FLASH DEAL`)
- **버튼 텍스트**: `X 상품 보기` / `X 혜택 보기` 형식, 4~8자 유지

### 현대카드 브랜드 보이스 3원칙

| 원칙 | 설명 |
|---|---|
| **자신감 있는** | ✏️ 세부 가이드 추가 예정 |
| **세련된** | ✏️ 세부 가이드 추가 예정 |
| **세심한** | ✏️ 세부 가이드 추가 예정 |

> 위 3원칙 기준으로 자동 생성 문구의 워딩을 검수합니다.

## 3. 이미지 생성 규격 (나노바나나2 API)

### 3-1. 기본 스펙
| 항목 | 값 |
|---|---|
| **비율** | 3 : 4 (세로형 모바일 기준) |
| **권장 픽셀** | 375 × 500 px (또는 배수) |
| **포맷** | JPG / PNG |
| **배경** | 메인 컬러 계열 단색 또는 미세 그라디언트 |

### 3-2. 오브젝트 레이아웃
```
┌─────────────────┐  ← 전체 높이 100%
│                 │
│   텍스트 영역    │  ← 상단 36% (텍스트 오버레이용, 요소 없음)
│                 │
├─────────────────┤
│                 │
│  ┌───────────┐  │
│  │  중심 오브젝트│  │  ← 높이 기준 64% 차지
│  │           │  │
│  └───────────┘  │
│                 │
└─────────────────┘  ← 하단
```

- 상단 **36%**: 완전 비워둠 (타이틀·날짜 텍스트 오버레이 영역)
- 중심 오브젝트: 전체 이미지 높이의 **64%** 차지, 하단 중앙 배치
- 오브젝트 좌우 여백: 최소 16px 이상

### 3-3. 스타일 지침
- 현대카드 브랜드 무드: clean, minimal, premium, modern
- 기하학적 또는 추상적 데코 요소 허용 (과도한 사실적 표현 지양)
- 텍스트, 로고, 타이포그래피 **절대 포함 금지**
- 메인 컬러 계열 내 미세 그림자·빛 효과 허용

### 3-3-1. 텍스트 문구 적용 불가 안내

> **이미지 생성 시 텍스트 문구(타이틀, 혜택 문구, 날짜 등)는 이미지에 반영되지 않습니다.**
>
> - 기획전 타이틀, 혜택 내용, 날짜 등 모든 텍스트는 이미지 위에 **오버레이(CSS 레이어)**로 표시됩니다.
> - 이미지는 순수 배경 비주얼(오브젝트·색상·분위기)만 생성합니다.
> - 프롬프트에 문구를 포함해도 AI가 이미지 내부에 텍스트를 렌더링하지 않도록 명시적으로 차단합니다.
> - **브랜드명("Hyundai", "Card", "M Mall", "M몰") 포함 일체의 한글·영문 문자가 이미지에 출력되지 않도록 프롬프트에 강제 금지 규칙을 명시합니다.**
> - AI 모델이 학습 데이터 기반으로 브랜드 텍스트를 자동 삽입하는 경우가 있으므로, 재생성 버튼으로 텍스트 없는 이미지가 나올 때까지 반복 생성을 권장합니다.

### 3-4. 나노바나나2 API 연동 방식

```
[호출 시점]
기획전 컨셉 입력 → "생성하기" 버튼 클릭 → 텍스트 생성과 동시에 이미지 프롬프트 생성 → API 호출

[프롬프트 구성 요소]
1. 기획전 타입 (시즌 / 플래시 / 세일 / 신규고객 등)
2. 메인 컬러 hex 값
3. 오브젝트 레이아웃 지침 (상단 36% 비움, 중심 오브젝트 64%)
4. 스타일 지침 (현대카드 무드)
```

### 3-5. 나노바나나2 API 상세

| 항목 | 값 |
|---|---|
| **엔드포인트** | `https://generativelanguage.googleapis.com/v1beta/models/imagen-3.0-generate-002:predict` |
| **인증키** | `?key=AIzaSyD3lAC9YuImKytAioFO80uoBv5aWTHOwBY` (쿼리 파라미터) |
| **요청 파라미터** | `instances[0].prompt` (프롬프트 문자열), `parameters.sampleCount` (1), `parameters.aspectRatio` ("3:4") |
| **응답 포맷** | **Base64 방식** — `predictions[0].bytesBase64Encoded` + `predictions[0].mimeType` |

### 3-6. 응답 이미지 바인딩 방식

```
[바인딩 흐름]
API 응답 수신 (Base64)
  └─ predictions[0].bytesBase64Encoded + predictions[0].mimeType
       └─ `data:image/png;base64,{b64}` 형태로 변환
            └─ dispatch(UPDATE) → hero 모듈 heroImage 필드 주입
                 └─ <img src={heroImage} style={{position:'absolute',inset:0,...}} />
                      → 히어로 배너 배경 전체 커버

[바인딩 대상 필드]
- 모듈 타입: hero
- 필드명: heroImage
- 적용 위치: 히어로 배너 배경 이미지 (position: absolute, objectFit: cover)

[로딩 UX]
- 생성하기 버튼: 비활성화 + "🎨 이미지 생성 중..." 텍스트
- 히어로 배너: 반투명 오버레이 + "이미지 생성 중..." 인디케이터
```

## 4. 모듈별 자동 생성 매핑 요약

| 모듈 | 생성 기준 |
|---|---|
| 🖼️ 히어로 배너 | 기획전 타입 → 서브타이틀 / 타이틀 / 날짜 |
| 📝 텍스트 섹션 | 타입 + 타깃 + 혜택 요약 → 2줄 설명 문구 |
| 🎟️ 쿠폰 | 혜택 키워드에서 원 단위 금액 추출, 타깃 조건 적용 |
| 🎁 혜택 리스트 | 혜택 키워드 파싱 결과 → 아이콘·제목·부제목 자동 구성 |
| 🛍️ 상품리스트 탭 | 카테고리 키워드 2개 이상 감지 시 탭 자동 생성 |
| 🔘 버튼 | 기획전 타입별 버튼 텍스트 |
| 📋 유의사항 | 기간 정보 삽입 + 공통 고정 문구 |
